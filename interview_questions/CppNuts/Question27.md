# C++ Shallow Copy vs. Deep Copy

C++에서 객체를 복사할 때, 멤버 변수가 **값(Value)** 인지 **포인터(Pointer)** 인지에 따라 복사 방식이 달라짐.

Shallow Copy (얕은 복사): 주소값만 복사하여 두 객체가 동일한 메모리를 가리킴. (위험!)

Deep Copy (깊은 복사): 새로운 메모리를 할당(new)하고 내부의 실제 값을 복사함. (독립적)

2. 코드 예시 (Deep Copy 구현)
```cpp
#include <iostream>

class Base {
    int* _ptr;  // 포인터 멤버 (Heap 할당 필요)
    int  _val;  // 일반 멤버 (Stack에 값 저장)

public:
    // 1. 기본 생성자
    Base() : _ptr{new int(0)}, _val{0} {}

    // 2. 깊은 복사 생성자 (핵심!)
    Base(const Base& rhs) {
        _ptr = new int;      // 별도의 메모리 공간 확보
        *_ptr = *rhs._ptr;   // 주소가 아닌 '실제 값'을 복사
        _val = rhs._val;     // 일반 변수는 그냥 복사 (항상 독립적)
    }

    // 3. 소멸자 (메모리 누수 방지)
    ~Base() { delete _ptr; }

    // Getter / Setter
    void setPtr(int v) { *_ptr = v; }
    void setVal(int v) { _val = v; }
    int getPtr() const { return *_ptr; }
    int getVal() const { return _val; }
};
```

3. 복사 방식 비교 분석
### 📌 일반 변수(`_val`) vs 포인터 변수(`_ptr`) 복사 비교

| 구분 | 일반 변수 (`_val`) | 포인터 변수 (`_ptr`) |
| :--- | :--- | :--- |
| **변수 타입** | `int` (데이터 직접 소유) | `int*` (데이터의 주소 소유) |
| **복사 대상** | `15`라는 숫자 값 자체 | 메모리 주소 (예: `0x123...`) |
| **복사 방식** | **항상 값 복사** (독립적) | **얕은 복사**: 주소 공유 <br> **깊은 복사**: 별도 메모리 할당(`new`) |
| **수정 시 영향** | `b2`를 바꿔도 `b1` 영향 없음 | **얕은 복사**: `b2` 수정 시 `b1`도 바뀜 <br> **깊은 복사**: 서로 영향 없음 |
| **메모리 위치** | 스택(Stack) 영역에 존재 | 실제 데이터는 힙(Heap) 영역에 존재 |

---

### 💡 핵심 포인트
1. **일반 변수(`_val`)**: 복사할 때 이미 새로운 공간에 값이 복제되므로 깊은/얕은 복사를 따질 필요가 없음. (항상 독립적)
2. **포인터 변수(`_ptr`)**: 주소값만 넘기느냐(`Shallow`), 주소가 가리키는 실제 값을 위해 새 집을 짓느냐(`Deep`)의 차이가 핵심.
3. **결론**: `15`가 유지되는 것은 C++의 기본 속성이고, **`10`이 유지되게 만드는 것** 이 우리가 구현한 '깊은 복사'의 목적입니다.

깊은 복사/얕은 복사 논쟁은 오직 **포인터(Heap 메모리 주소)** 를 가진 멤버에서만 발생.

결론: 일반 변수가 독립적인 건 당연한 것이고, 포인터 변수까지 독립적으로 만든 것이 '깊은 복사'의 핵심.

**Rule of Three** 

실무에서 포인터를 사용하는 클래스를 설계할 때는 다음 3가지를 반드시 세트로 구현해야 함.

Destructor (소멸자): delete 안 하면 메모리 누수 발생.

Copy Constructor (복사 생성자): 안 만들면 얕은 복사 발생.
Copy Assignment Operator (대입 연산자): b2 = b1 시 얕은 복사 방지.


### 추가 설명
1) 포인터가 없으면 Rule of Three를 지킬 필요가 없다.
C++에서 `int a`나 `std::string s;` 같은ㅇ 일반적인 변수만 쓰는 클래스는 소멸자, 복사 생성자, 대입 연산자를 안 만들어도됌.
- 이유: 컴파일러가 기본으로 만ㄷ르어주는 애들이 미 충분히 똑똑하게 (값 복사로) 잘 작동.
- Rule of Zero: "아무것도 직접 만들지 마라" 라는 원칙이 적용되는 상황.

2) 포인터(`new`)가 있으면 **Rule of Three** 는 무조건 세트다.
    - 소멸자: `new` 했으니까 `delete` 해야 함. (안 하면 메모리 누수)
    - 복사 생성자: `Base b2 = b1;` 할 때 새 메모리 할당해야 함. (안 하면 얕은 복사)
    - 대입 연산자: `b2 = b1;` 할 때 기존 메모리 지우고 새로 할당해야 함. (안 하면 메모리 누수 + 얕은 복사)
