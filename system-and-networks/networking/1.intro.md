### Internet
- The internet is a group of many networks that all use the **IP protocol**.
- A host is any computer connected to the network.
- Routers and switches only forward packets.(That's literally it)
- All real "intelligence" lives in the **host's protocol stack**, not int the network.
-> This is known as the **End-to-End Principle**. It'sthe reason the Internet scaled so well. However, in modern engineering (especially MCP or Low-latency systems), we noe use "Smart NICs" or "Programmable Switches" (P4) that do have intelligence.
- The Pro View is to acknowledge the End-to-End principle, but know that "Middleboes" (Load Balancers, Firewalls) break this rule to provide security and performance.

#### Packet
- **Packet = Header + Payload**
    - Header: addresses and control information 
    - Payload: the actual data
- A packet is usually about 1500 bytes, so large messages are split into many packets.
-> 1500 bytes is the **Standard Ethernet MTU (Macimum Transmission Unit).** In high-performance environment, we use **Jumbo Frames**(9000 bytes. If you're moving large GNN weights, for example, using 1 500-byte packets creates massive CPU overhead because the OS has to process 6x as many headers.

#### How Communication Works
1. Sender(p1)
    - The application creates a message.
    - The protocol stack breaks it into packets and adds headers.
    - Packets are sent out into the network.
2. Inside the Network
    - Routers read the header and forward the packet toward the destination.
    - They do not look at the data or try to rebuild the message.
3. Receiver (P2)
    - The protocol stack receives the packets
    - Headers are removed, packets are put back in order, and the original message is rebuilt.
    - The message is given to the application.

#### Key features of IP
- **Packet-switched network:** A message is split into many small packets.
- **Best-effort delivery:** No guarantee of delivery, order, or reliability.
- **Routing:** Each packet chooses its own path independently.

#### Protocol Stack
- **Definition: TCP/IP Stack**
    - Application(7): HTTP, FTP, JSON-RPC, etc. (creates messages)
    - Transport(L4):
         - TCP: Reliable, Sequential. Adds Ports, Sequence Numbers (for ordering), and handles Segmentation.
         - UDP (Fire-and-Forget): Connectionless, Fast. Adds Ports only; no ordering, no retransmission (minimal overhead). The order of packets might get messed up.
    - Network(L3): IO (packet forwarding. Packetizing, adds IP address (for host-to-host routing))
    - Link(L2): Ethernet, Wi-Fi (Framing, adds MAC addresses (for physical hop-to-hop movement))

#### Why IP Needs a Protocol Stack
IP only forwards packets. It cannot solve problems like:
- Large messages (need splitting and reassembly)
- Out-of-order packets
- Lost packets
- Bit errors during transmission
- Congestion and queuing delays

-> The protocol Stack adds reliability, ordering, error recovery, and flow control at higher layers.
